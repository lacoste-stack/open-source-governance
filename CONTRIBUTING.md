# Contribution Guidelines  

## Getting Started  
1. Fork the repository and clone your fork locally.  
2. Install dependencies: `npm install` (or `pip install -r requirements.txt`, etc.).  
3. Create a branch: `git checkout -b feat/my-feature`.  

## Code Style  
- Follow [PEP 8](https://peps.python.org/pep-0008/) for Python.  
- Use ESLint/Prettier for JavaScript.  
- Write meaningful commit messages (e.g., `feat: add login button`).  

## Pull Request Process  
1. Ensure tests pass: `npm test` (or `pytest`, etc.).  
2. Update documentation (README, comments, etc.).  
3. Open a PR against the `main` branch.  
4. **Requirements**:  
   - 2 approvals from maintainers.  
   - Link to an issue using `Closes #123`.  

## Reporting Issues  
- Use the issue template (see [ISSUE_TEMPLATE.md](./.github/ISSUE_TEMPLATE.md)).  
- Include steps to reproduce, OS/environment, and screenshots.  

## Code of Conduct  
- Adhere to the [Code of Conduct](./CODE_OF_CONDUCT.md).
